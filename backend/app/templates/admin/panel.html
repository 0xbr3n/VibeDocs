🔑 Password actions —

Pick one of the two flows below. Both are audit-logged. Force-set is best when the user is locked out and you want to hand them a temporary password directly. The reset-link path is the standard "self-serve" flow — the user picks their own new password after clicking the email link.

Option 1 — Force-set a new password

The new password is hashed + saved immediately. Any in-flight reset link for this account is invalidated. A confirmation email is sent to the user notifying them of the change (this bypasses their notification preference because it's a security event).

Option 2 — Email a reset link

A one-time reset URL (valid for 30 minutes) is mailed to —. The user follows the link and picks their own password. Any prior unused reset tokens on the account are invalidated when the new one is issued.
