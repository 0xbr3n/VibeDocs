📝 Template Instructions

Open your Word document and drop these placeholders wherever you want the data to appear. The generator replaces each one when a report is built.

⚠ Copy only the green text — just the {{ … }} part. Do NOT include any {% raw %} wrappers in your template; those prevent the placeholder from being substituted at render time.

Project metadata

  • {{ project.client_name }} — Client / company name
  • {{ project.name }} — Project name
  • {{ project.testing_window }} — Test dates (e.g. 2026-05-01 to 2026-05-10)
  • {{ project.scope_description }} — Scope details (free text)
  • {{ project.sector }} — Sector (Finance, Retail, Healthcare, etc.)

Report metadata

  • {{ report.name }} — Report name
  • {{ report.version }} — Version (e.g. 0.1)
  • {{ generated_at }} — Build timestamp (UTC)

Master prose sections (edit under Templates → Master prose)

  • {{ sections.executive_summary }} — Executive summary
  • {{ sections.methodology }} — Testing methodology
  • {{ sections.scope_disclaimer }} — Scope / disclaimer

Findings (rows + per-finding detail)

  • {{ total_findings }} — Total finding count
  • {{ severity_counts.Critical }}, {{ severity_counts.High }}, …, {{ severity_counts.Informational }} — Counts per severity
  • {{ severity_chart }} — Auto-generated severity bar chart (image)
  • Findings table row loop (place inside the table row):
    {%tr for f in findings %} {{ loop.index }} | {{ f.title }} | {{ f.severity }} | {{ f.cvss_score }} | {{ f.status }} {%tr endfor %}
  • Per-finding detail paragraph loop:
    {%p for f in findings %} Finding {{ loop.index }}: {{ f.title }} Severity: {{ f.severity }} ({{ f.cvss_score }}) Affected: {{ f.affected_asset }} {{ f.description }} {{ f.impact }} {{ f.remediation }} {%p endfor %}

✓ The generator uses Jinja2 syntax — double curly braces wrap each placeholder.
✓ Place them in headers, body, tables, or footers — all are scanned.
✓ Required at minimum: {{ project.client_name }} and the findings row loop.
✓ Missing optional placeholders fall back to defaults — they won't break the build.
✓ If a placeholder you typed renders as literal text instead of being substituted, double-check spelling against the list above, and make sure you didn't bring any {% raw %} wrappers along with you.
