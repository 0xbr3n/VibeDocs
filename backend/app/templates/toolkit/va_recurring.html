What's in the ZIP

  • remaining_findings.xlsx — current scan minus anything matched to a risk-accepted item, categorised.
  • risk_accepted_removed.xlsx — what was subtracted, with evidence columns (source, keyword, comment cell, comment text).
  • by_category/*.xlsx — one workbook per category (Outdated Software & Patches, SSL Misconfigurations, Information Disclosure, Default / Weak Credentials, Web Application, Uncategorized). Friendly column names + writable Status / Comments.
  • considered_not_removed.xlsx — accepted-side rows the matcher considered but didn't subtract, plus near-miss diagnostics so you can triage "why didn't this drop".
  • audit_remaining_vs_accepted.xlsx — suspect rows in the remaining set that have an accepted near-miss on the same host (catches column-mapping mistakes on the accepted side).
  • match_preview.xlsx — borderline subtractions for spot-checking.
  • summary.txt — top-level numbers + category counts.

The plugin_id_categories.json learning map is shared across every consultant who runs the tool — high-confidence categorisations from previous runs are skipped at the rule-engine step. Edit by hand to correct mistakes (it lives under uploads/toolkit/va_recurring/ on the server).
